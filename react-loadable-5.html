React Loadable 5

Code Splitting + Server Side Rendering Made Easy

This article is best enjoyed while listening to: "Joy" – Levin Kali

If you're unfamiliar, React Loadable is a small library designed to make it easy to code split your React app whereever you render a component (everywhere). It looks like this: 

import Loadable from 'react-loadable';
import Loading from './my-loading-component';

const LoadableComponent = Loadable({
  loader: () => import('./my-component'),
  loading: Loading,
});

export default class App extends React.Component {
  render() {
    return <LoadableComponent/>;
  }
}

Notice the import(), this is a new feature in JavaScript currently at Stage 3 of the standardization process. Instead of writing your imports at the top of your file, you can use import() anywhere and it will return a promise that will resolve with your module.

Webpack makes use of import() to "code split" your bundles automatically. This means instead of serving one massive bundle, you can serve many small ones. 

import express from 'express';
import React from 'react';
import ReactDOMServer from 'react-dom/server';
import Loadable from 'react-loadable';
import App from './components/App';

const app = express();

app.get('/', (req, res) => {
  res.send(`
    <!doctype html>
    <html>
      <head>
        <title>My App</title>
      </head>
      <body>
        <div id="app">
          ${ReactDOMServer.renderToString(<App/>)}
        </div>
      </body>
    </html>
  `);
});

Loadable.preloadAll().then(() => {
  app.listen(3000, () => {
    console.log('Running on http://localhost:3000/');
  });
});
